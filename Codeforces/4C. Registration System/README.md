## 문제 설명

새로운 이메일 서비스 "Berlandesk"가 가까운 미래에 Berland에서 열릴 예정입니다. 사이트 관리자는 가능한 한 빨리 프로젝트를 시작하고 싶어하므로, 여러분에게 도움을 요청합니다. 여러분은 사이트 등록 시스템의 프로토타입을 구현하는 제안을 받았습니다. 시스템은 다음 원칙에 따라 작동해야 합니다. 새로운 사용자가 등록을 원할 때마다, 그는 자신의 $name$을 시스템에 요청으로 보냅니다. 만약 그러한 $name$이 시스템 데이터베이스에 존재하지 않는다면, 데이터베이스에 삽입되고 사용자는 성공적인 등록을 확인하는 응답인 $OK$를 받습니다. 만약 $name$이 이미 시스템 데이터베이스에 존재한다면, 시스템은 새로운 사용자 이름을 만들어 사용자에게 제안으로 보내고, 제안된 이름도 데이터베이스에 삽입합니다. 새로운 이름은 다음 규칙에 따라 형성됩니다. 숫자 $1$부터 시작하여 $name$에 차례로 붙여 $name1$, $name2$, ...와 같이 되며, 이 숫자들 중에서 가장 작은 $i$를 찾아 $name_i$가 데이터베이스에 아직 존재하지 않는 경우를 찾습니다.

## 입력

첫 번째 줄에는 숫자 $n$ ($1 \le n \le 10^5$)이 포함되어 있습니다. 그 다음 $n$줄에는 시스템에 대한 요청이 포함되어 있습니다. 각 요청은 비어 있지 않은 한 줄로, 모두 소문자 라틴 문자로 이루어져 있으며, 최대 32자 이하입니다.

## 출력

n개의 줄을 출력하시오. 각 줄은 요청에 대한 시스템 응답으로, 성공적으로 등록된 경우에는 OK를 출력하고, 요청한 이름이 이미 사용 중인 경우에는 새로운 이름을 제안하는 메시지를 출력한다.
